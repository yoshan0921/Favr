Volunteer Dashboard

This page will be the dashboard for the volunteer and as we saw on the wireframes, is on this page that the volunteer will see their tasks. The app will need to get data from the "Tasks" collection in the database. I've added some dummy tasks in our database, fetched them through the API and displayed them on the table below, just like the JS exercises we did last term.

Examine the code that does this (starting from line 56 of js/volunteers/home.js) and see how I use the getAll function from the firestore.js file. This file is where all the communication with the database will happen. Besides the getAll function, this file has functions to create, update (edit), and delete a document. You don't need to understand everything that is going on in this file. Just read the documentation I wrote and trust it lol.

You can practice your understanding of the firebase API by making the form next to the table work. I've written pretty much everything for you. All you need to worry about is studying the code and figuring out how to use the functions of the firestore.js file on the form submission. Look closely at how I used the getAll function. The procedure to call other functions is the same.

By the way, always check the console. Error messages are useful for understanding why things aren't working

You can view our database in the Firebase Console: Build > Firestore Database
